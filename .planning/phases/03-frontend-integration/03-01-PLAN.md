3. CI: Generate and commit during release

Recommended approach per D-01: CLI command for build-time generation.

  <name>Task 4: Create CLI command for OpenAPI spec generation</name>
  <files>
    packages/clerk/src/openclerk/cli/commands/docs.py

  <action>
    Create packages/clerk/src/openclerk/cli/commands/docs.py with OpenAPI generation command:

    ```python
    """Documentation commands for Clerk CLI."""

    import json
    from pathlib import Path

    import typer
    from fastapi.openapi.utils import get_openapi

    from ...web.app import create_app

    app = typer.Typer()


    @app.command("generate-openapi")
    def generate_openapi(
        output: Path = typer.Option(
            Path("openapi.json"),
            "--output",
            "-o",
            help="Output file path",
        ),
    ) -> None:
        """Generate OpenAPI specification from FastAPI app."""
        app_instance = create_app()
        openapi_schema = get_openapi(
            title=app_instance.title,
            version=app_instance.version,
            openapi_version=app_instance.openapi_version,
            description=app_instance.description,
            routes=app_instance.routes,
        )

        output.parent.mkdir(parents=True, exist_ok=True)
        with open(output, "w") as f:
            json.dump(openapi_schema, f, indent=2)

        typer.echo(f"OpenAPI spec written to {output}")
    ```

    Then register the command in packages/clerk/src/openclerk/cli/main.py by adding:
    - Import: `from .commands import docs`
    - Registration: `app.add_typer(docs.app, name="docs")`

    Per D-01, this generates static openapi.json for serving.

  <name>Task 2: Create environment variables reference</name>
  <files>docs/deployment/environment.md</files>
  <action>
Create `docs/deployment/environment.md` — Complete environment variables reference (per DEPLOY-02):

Structure:
1. **Overview** — How environment variables work
   - .env file mounting (per D-04)
   - Variable substitution in docker-compose.yml
   - Security best practices (file permissions)

2. **Required Variables**
   - DATABASE_URL — PostgreSQL connection string
     - Format: `postgresql://user:pass@host:port/db`
     - Example for Docker: `postgresql://clerk:clerk@db:5432/clerk`
   - OPENAI_API_KEY — LLM provider API key
     - Required for AI reasoning workflows
     - Get from https://platform.openai.com

3. **Application Variables**
   - SECRET_KEY — Session signing key
     - Generate with: `openssl rand -hex 32`
     - Must be kept secret in production
   - LOG_LEVEL — Application logging verbosity
     - Options: debug, info, warning, error
     - Default: info

4. **Optional Integrations**
   - SUPABASE_URL — Supabase project URL
   - SUPABASE_KEY — Supabase anon/public key
   - SUPABASE_SERVICE_KEY — Supabase service role key
   - When to use Supabase vs local PostgreSQL

5. **Docker Compose Variables**
   - POSTGRES_USER — Database username (default: clerk)
   - POSTGRES_PASSWORD — Database password (default: clerk)
   - POSTGRES_DB — Database name (default: clerk)
   - FRONTEND_PORT — Host port for frontend (default: 80)

6. **Frontend Variables**
   - VITE_API_URL — Backend API URL for frontend
     - Default: http://localhost:8000
     - Change for production deployment

7. **Security Considerations**
   - Never commit .env to version control
   - Use strong SECRET_KEY in production
   - Protect .env file permissions (chmod 600)
   - Rotate API keys regularly

8. **Quick Reference Table**
   - Markdown table with: Variable | Required | Default | Description

Reference variables from Plan 04-01's .env.example file.
